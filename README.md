# FC Barcelona Legends Web Page\n\nA Next.js 14 application showcasing legendary FC Barcelona players, powered by Adobe AEM Content Fragments as a headless CMS.\n\n## Project Overview\n\nThis application renders a dynamic web experience featuring:\n- **Legends listing page** with player cards\n- **Individual legend detail pages** with full biography and achievements\n- **Trophy showcase** integrated with player achievements\n- **Hero section** with configurable headline and subtext\n\nAll content is authored and managed in Adobe AEM Content Fragments, served via GraphQL to this Next.js frontend.\n\n## Prerequisites\n\n- **Node.js** 18 or higher\n- **npm** 9 or higher (or yarn/pnpm)\n- **AEM Publish Instance** with GraphQL endpoint configured\n- **Bearer token** for AEM authentication (if required by your tenant)\n\n## Setup Instructions\n\n### 1. Clone the Repository\n\n```bash\ngit clone https://github.com/kacemlight/barca-legends.git\ncd barca-legends\n```\n\n### 2. Install Dependencies\n\n```bash\nnpm install\n```\n\n### 3. Configure Environment Variables\n\nCopy `.env.local.example` to `.env.local` and fill in your AEM details:\n\n```bash\ncp .env.local.example .env.local\n```\n\nEdit `.env.local`:\n\n```\nNEXT_PUBLIC_AEM_HOST=https://publish-p<program>-e<env>.adobeaemcloud.com\nNEXT_PUBLIC_AEM_GRAPHQL_ENDPOINT=/content/graphql/global/endpoint.json\nNEXT_PUBLIC_AEM_AUTH=<bearer-token-if-needed>\n```\n\n**Environment Variables:**\n\n- `NEXT_PUBLIC_AEM_HOST` ‚Äî Your AEM publish instance URL (must be HTTPS)\n- `NEXT_PUBLIC_AEM_GRAPHQL_ENDPOINT` ‚Äî Path to the AEM GraphQL endpoint (default: `/content/graphql/global/endpoint.json`)\n- `NEXT_PUBLIC_AEM_AUTH` ‚Äî Bearer token for authenticated requests to AEM (optional, leave blank if public access)\n\n### 4. Run Development Server\n\n```bash\nnpm run dev\n```\n\nOpen [http://localhost:3000](http://localhost:3000) in your browser. The page will auto-refresh as you edit files.\n\n### 5. Build for Production\n\n```bash\nnpm run build\nnpm run start\n```\n\n## Project Structure\n\n```\nbarca-legends/\n‚îú‚îÄ‚îÄ app/\n‚îÇ   ‚îú‚îÄ‚îÄ layout.tsx              # Root layout with *(string, required)* ‚Äî Full player name\n- `nickname` *(string, optional)* ‚Äî Player nickname\n- `position` *(enum)* ‚Äî `goalkeeper` | `defender` | `midfielder` | `forward`\n- `era` *(string)* ‚Äî Playing period (e.g., \"2004‚Äì2021\")\n- `bio` *(string, long-text)* ‚Äî Biography and career highlights\n- `trophies` *(string[], multi-value)* ‚Äî List of trophy titles won\n- `photo` *(content-reference)* ‚Äî Reference to DAM image asset\n- `nationality` *(string)* ‚Äî Player's country\n- `appearances` *(number)* ‚Äî Total appearances for club\n- `goals` *(number)* ‚Äî Total goals scored\n\n### Trophy (Model: `trophy`)\n\nRepresents a competition trophy.\n\n- `title` *(string, required)* ‚Äî Trophy name\n- `year` *(number, required)* ‚Äî Year won\n- `competition` *(enum)* ‚Äî `la-liga` | `copa-del-rey` | `uefa-champions-league` | `fifa-club-world-cup` | `supercopa` | `uefa-super-cup`\n\n### PageConfig (Model: `page-config`)\n\nGlobal page configuration.\n\n- `pageTitle` *(string)* ‚Äî HTML page title\n- `heroHeadline` *(string)* ‚Äî Hero section headline\n- `heroSubtext` *(string)* ‚Äî Hero section subtext\n- `metaDescription` *(string)* ‚Äî Meta description for SEO\n\n## AEM GraphQL Integration\n\n### Endpoint Details\n\nThis app queries the AEM GraphQL endpoint at:\n\n```\nhttps://{NEXT_PUBLIC_AEM_HOST}{NEXT_PUBLIC_AEM_GRAPHQL_ENDPOINT}\n```\n\nFor sandbox tenant `acssandboxemea02jcadev`, the full URL is:\n\n```\nhttps://publish-p<program>-e<env>.adobeaemcloud.com/content/graphql/global/endpoint.json\n```\n\n### GraphQL Queries\n\nSee `lib/queries.ts` for:\n- `GET_ALL_LEGENDS` ‚Äî Fetch all legend fragments\n- `GET_LEGEND_BY_SLUG` ‚Äî Fetch a single legend by name/slug\n- `GET_ALL_TROPHIES` ‚Äî Fetch all trophy fragments\n- `GET_PAGE_CONFIG` ‚Äî Fetch page configuration\n\n### Making Requests\n\nThe `lib/aem.ts` client handles all GraphQL requests. Example:\n\n```typescript\nimport { getAllLegends } from '@/lib/aem';\n\nconst legends = await getAllLegends();\n```\n\n## Error Handling & Loading States\n\nAll page components gracefully handle:\n- **Loading states** ‚Äî Display skeleton or spinner while fetching data\n- **Error states** ‚Äî Show user-friendly error messages if data fetch fails\n- **Empty states** ‚Äî Handle cases where no content is available\n\nNo silent failures ‚Äî all errors are logged to console and displayed in UI.\n\n## Technologies\n\n- **Next.js 14** ‚Äî React meta-framework with App Router\n- **TypeScript** ‚Äî Static type checking\n- **graphql-request** ‚Äî Minimal GraphQL client\n- **AEM Content Fragments** ‚Äî Headless CMS backend\n\n## Available Scripts\n\n- `npm run dev` ‚Äî Start development server on http://localhost:3000\n- `npm run build` ‚Äî Build for production\n- `npm run start` ‚Äî Start production server\n- `npm run lint` ‚Äî Run ESLint\n\n## Deployment\n\nThis app is ready to deploy to Vercel, Netlify, or any Node.js-compatible hosting:\n\n1.
